# LinguaChat (语伴) — AI 外语伴学工具

> 一个专为外语学习者设计的 AI 聊天桌面应用。
> 通过和 AI 角色自由对话，在沉浸式场景中自然习得外语。

---

## 项目概述

**核心理念**：语言是在使用中学会的，不是被教会的。LinguaChat 让用户通过和 AI 角色在**多场景群聊**中自由交流来学习外语，AI 同时扮演"聊天伙伴"和"语言老师"双重角色。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 🤖 **AI 对话** | 支持 OpenAI / Anthropic / OpenRouter / Ollama 等主流 API |
| 👥 **多角色系统** | 用户可创建多个 AI 角色（场景、性格、语气），每个角色独立设定 |
| 🗣️ **群聊模式** | 多个 AI 角色 + 用户在同一个聊天室对话，话题自然展开 |
| 🌐 **双语回复** | AI 每次回复同时生成"目标外语 + 母语"（母语可开关） |
| 👆 **单词点击** | 点击外语句子中的任意单词 → 显示释义 + 发音 |
| 🔊 **语音朗读** | 每个单词/句子可朗读标准发音（调用 Google TTS 或系统 TTS） |
| ✏️ **纠错模式** | 用户用外语表达有误时，AI 先纠正再继续对话 |
| 🎯 **渐进式学习** | 用户从"母语聊天 → AI双语回复"逐步过渡到"外语聊天 → 仅外语回复" |
| ⚙️ **灵活设置** | API 配置、语言对、难度级别、母语显示开关等 |

## 方案分析

### 方案对比：基于 SillyTavern 修改 vs. 从零构建

| 维度 | 基于 SillyTavern 修改 | ✅ **从零构建（推荐）** |
|------|----------------------|----------------------|
| **开发速度** | 初期快（已有聊天UI + API集成） | 初期慢（需搭建基础架构） |
| **代码复杂度** | 极高（30万+行，jQuery 旧代码） | 可控（按需编写） |
| **功能匹配度** | 低（角色扮演游戏导向，需大量删改） | 高（专为语言学习设计） |
| **长期维护** | 难（上游更新需合并冲突） | 易（完全自控） |
| **打包为 macOS App** | 需额外 Electron 封装 | 原生 Electron/Tauri 支持 |
| **扩展性** | 受限于插件系统边界 | 无限 |

**结论**：SillyTavern（酒馆）虽然功能强大，但它本质是为"角色扮演聊天"设计的，其世界书、AI 绘图、群聊等功能对语言学习场景来说是冗余的。其前端基于 **Vanilla JS + jQuery**，没有现代化的组件化架构，做深度定制反而更困难。

**推荐方案**：使用 **Electron + React + Vite + TypeScript** 从零构建，轻量、可控、易维护。

多个 AI 角色 + 用户在同一个聊天室对话，话题自然展开。

```
│  ┌──────────────────────────┐            │

│  └──────────────────────────┘            │
│                                          │
│  🧑 Emma · 咖啡师                        │

│  │ 🇬🇧 Hey there! Yeah,     │            │

│  │    finished. 😊           │            │

│  [你: 听起来很棒！你们有没有招牌...]  ➤  │
├──────────────────────────────────────────┤

│  [+ 添加角色]  [💬 切换到1对1]            │
└──────────────────────────────────────────┘
```

#### 群聊的优势

1. **场景自然**：咖啡馆、办公室、课堂等场景天然适合多人对话
2. **话题自动延伸**：角色之间会互相回应，让对话像真实交谈一样自然流动
3. **不同的语言风格**：正式/非正式、专业/日常、不同口音和表达习惯
4. **观察学习**：用户可以听角色之间的对话学习，再参与进去
5. **角色轮换**：同样的场景换不同角色组合，获得不同话题

#### 双模式切换

```
┌──────────┐
│ 1对1模式 │  ←→  │ 群聊模式 │
──────────┘      └──────────
 用户 ↔ AI角色        用户
                  ↙   ↓   ↘
               AI角1 AI角2 AI角3
```

### 4. 单词点击 (Word Click)

用户点击外语句子中的任意单词：

```
┌───────────────┐
│    walked     │
│  ───────────  │
│  /wɔːkt/      │ ← 音标
│  步行，散步    │ ← 中文释义
│  🔊 朗读       │ ← 发音按钮
│  📋 加入生词本  │
└───────────────┘
```

采用 **Google Translate API** 免费接口获取单词释义和发音，无需注册 Key。

### 5. 纠错机制 (Error Correction)

当用户尝试用外语表达但存在语法/用词错误时：

1. AI 识别出错误
2. 用温和的方式指出正确表达（✏️ 标记）
3. 给出简短解释
4. 继续正常的对话回复（包含外语 + 母语）

**系统 Prompt 设计示例**：
```
你是一位外语教师兼聊天伙伴，角色是 [角色名] — [角色描述]。

你的任务是与用户进行自然对话，同时帮助用户学习 [目标语言]。

规则：
1. 请严格按照你的人物设定（身份、性格、语气）来回复
2. 如果当前是群聊模式，请与其他 AI 角色自然互动
3. 用户可以使用 [母语] 或 [目标语言] 与你交流
4. 每次回复你都要输出 [目标语言] 版本，并在需要时附上 [母语] 翻译
5. 如果用户使用 [目标语言] 时出现错误，先用 ✏️ 温和地纠正，然后继续对话
6. 纠正要简短，不要打断对话流畅性
7. 根据用户的水平调整语言复杂度
8. 母语辅助翻译可以通过设置开启/关闭
```

### 6. 渐进式学习路径

```
阶段 1: 适应期
  用户 → 母语聊天
  AI   → 外语 + 母语

阶段 2: 模仿期
  用户 → 母语夹带简单外语
  AI   → 外语 + 母语（开始纠错）

阶段 3: 尝试期
  用户 → 主要用外语，必要时母语
  AI   → 外语 + 母语（持续纠错）

阶段 4: 熟练期
  用户 → 全外语
  AI   → 仅外语（用户可选是否显示母语）
```

## 简单实现技巧（让开发更容易）

1. **Google Translate 免费接口**：使用 `https://translate.googleapis.com/translate_a/single?...` 无需注册 API Key，直接获取单词释义和发音
2. **Web Speech API**：浏览器内置的 `SpeechSynthesisUtterance` 可实现 TTS，无需额外服务
3. **系统 Prompt 驱动**：所有语言学习逻辑通过精心设计的 System Prompt 实现，无需复杂的后端逻辑
4. **Zustand 状态管理**：比 Redux 简单得多，几行代码搞定全局状态
5. **Tailwind CSS**：快速构建聊天 UI，无需手写大量 CSS
6. **角色系统即数据**：角色信息存储为 JSON 文件，无需数据库
7. **群聊即多轮 API 调用**：群聊通过轮询顺序调用各角色 API，每次带上完整上下文
