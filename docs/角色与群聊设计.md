# 角色系统与群聊模式 — 补充分析

## 一、为什么角色+群聊是语言学习的利器

你提的这个需求非常好，它解决了 AI 外语学习中最关键的**场景缺失**问题：

### 只用单角色聊天的问题
```
用户: "今天天气怎么样？"
AI:   "今天天气很好。你周末有什么计划？"
用户: "我打算去购物。"
AI:   "听起来不错！你喜欢去哪里购物？"
    ⋮ 对话容易变无聊、话题枯竭
```

### 多角色群聊的效果
```
用户: "今天天气怎么样？"

Emma(咖啡师): "阳光明媚！这种天气我的咖啡店露台
              坐满了人 ☀️"

Marco(厨师): "Perfect weather for gelato!
              知道吗，罗马的夏天..."
              (话题自然延伸)

Emma → Marco: "哦，Marco 总是能把任何话题引到美食上！😂"
              (角色间互动产生新话题)

用户: 学到"gelato"这个新词，也学到了如何谈论天气
```
**群聊 = 动态场景 + 多风格输入 + 自然话题流**

## 二、角色系统设计

### 角色的数据模型

```typescript
interface Character {
  id: string;
  name: string;              // 角色名
  age?: number;               // 年龄
  avatar?: string;            // 头像（Emoji/图片路径）
  occupation: string;         // 身份/职业
  nativeLanguage: string;     // 角色母语
  personality: string;        // 性格描述
  background: string;         // 人物小传（背景故事）
  speakingStyle: string;      // 说话风格
  interests: string[];        // 兴趣爱好
  sceneTags: string[];        // 场景标签，如 ["咖啡厅","日常"]
  languageLevel: string;      // 角色使用目标语言的水平
}
```

## 四、群聊的技术实现方案

群聊看起来复杂，但实现方式其实很简单：

### 核心思路：顺序轮询 API

```
用户发送消息
    │
    ▼
将完整对话历史 + 用户消息 发送给 角色1 (Emma) 的 API
    │ ← 角色1 回复
    ▼
将完整对话历史 + 角色1的回复 发送给 角色2 (Marco) 的 API
    │ ← 角色2 回复
    ▼
两条回复一起渲染到界面
    │
    ▼
等待用户下一条输入
```

**关键点**：
- 每次只调一个角色的 API，串行执行
- 每个角色看到的是**完整的对话历史**（包括其他角色的发言）
- 所以角色之间会"看到"彼此说了什么，从而产生互动
- API 调用可以用流式（streaming）逐字显示，也可以用非流式一次返回

### 为什么这样简单？

不需要复杂的多智能体通信协议，不需要管理对话树，不需要角色间协商——大模型本身就具备"理解上下文"和"扮演角色"的能力。你只需要：

1. 把完整的对话历史传给 API
2. 在 System Prompt 里告诉它"你是 Emma，你看到 Marco 刚说了..."
3. AI 就会自然地接话

## 七、群聊时的纠错策略

群聊中纠错需要更自然的方式，不能让角色显得"爱说教"：

```
策略: 角色不做"老师式纠错"，而是「自然重述」

用户: "Yesterday I go to park with friend."

Emma: "Oh, you WENT to the park? That's lovely!
       ☕ Here's your usual latte, by the way.
       Where did you and your friend go exactly?"

Marco: "Ah, the park! In Rome we say 'andare al parco'.
        Did you have a nice walk?"

关键: Emma 自然地用 "WENT" 重述了用户的错误，
     但重点在继续对话，而不是停下来上课。
```

**纠错的三种模式**：

| 模式 | 适用场景 | 方式 |
|------|---------|------|
| 🟢 **自然重述** | 群聊模式 | 用正确说法自然地接话，不直接指出错误 |
| 🟡 **温和提示** | 1对1模式 | ✏️ 简短提示正确说法，然后继续 |
| 🔴 **教学模式** | 用户主动请求 | 详细解释语法规则+举例 |
